# Performance Review Playbook

## Review order

1. Per-frame work.
2. Event connection leaks.
3. Expensive workspace scans.
4. UI update frequency.
5. Physics assemblies and network ownership.
6. Too many parts/effects.
7. DataStore/write frequency.
8. Streaming assumptions.

## Output format

- Highest risk issue.
- Quick wins.
- Structural fixes.
- Profiling/test steps.
