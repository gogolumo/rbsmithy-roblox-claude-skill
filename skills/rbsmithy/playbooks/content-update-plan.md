# Content update plan playbook

Use for weekly/monthly updates, events, or new content drops.

## Steps
1. Define update goal.
2. Pick one player-facing promise.
3. Scope systems and assets.
4. Add analytics.
5. Add QA checklist.
6. Add rollback plan.
7. Draft release notes.

## Pushback rule
If the update tries to add too many systems at once, cut scope to the smallest playable slice.
