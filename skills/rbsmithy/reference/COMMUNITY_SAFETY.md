# Community safety and moderation

Use this file for chat, custom text, player reports, guild/crew names, ship names, signs, usernames, display names, user-generated content, and social systems.

## Rules

- Do not display user-generated text without Roblox text filtering.
- Do not build systems that bypass Roblox chat or safety systems.
- Do not store raw unsafe chat content unless the system and policy allow it.
- Avoid exposing minors to unrestricted cross-age communication systems.
- Provide player reporting flows for social features.
- Keep moderation tools server-authoritative.
- Do not place moderation secrets or admin keys in client-accessible objects.

## Systems that need review

- custom ship names;
- guild/crew names;
- player bios;
- signs and notes;
- chat commands;
- trading messages;
- party finder descriptions;
- marketplace listings;
- roleplay text.

## Safe design alternatives

- preset message wheels;
- filtered text input;
- limited templates;
- moderation queues;
- report buttons;
- block/mute tools;
- server-side audit logging.

## Claude output requirements

For social or UGC features, include:

1. UGC risk classification;
2. text filtering plan;
3. report flow;
4. moderation storage plan;
5. abuse cases;
6. Studio tests;
7. compliance warnings.
