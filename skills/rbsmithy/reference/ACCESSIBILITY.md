# Accessibility

Use this reference for UI, controls, combat feedback, tutorials, dialogue, and audio/visual cues.

## Accessibility checklist

- readable text size on mobile;
- sufficient contrast between text and background;
- important information is not color-only;
- audio cues have visual alternatives;
- visual cues have text/icon alternatives where needed;
- buttons have touch-friendly size;
- gamepad navigation is possible;
- time-limited prompts are not too short;
- camera shake can be reduced/disabled for sensitive players;
- subtitles or text versions exist for important voice/audio information.

## UI rules

- Do not make tiny buttons because they look cool on desktop.
- Avoid low-contrast parchment/gold text combinations.
- Use icons plus text for important actions.
- Make focus/selection visible for gamepad.
- Keep critical HUD outside topbar/notch unsafe zones.

## Game design rules

Accessibility is not only UI. Also review:

- enemy telegraphs;
- quest directions;
- combat impact feedback;
- navigation markers;
- colorblind-safe rarity systems;
- readable maps/minimaps;
- skip/recap options for long dialogue.
