# Game design documents and MVP scope

Use this file when turning ideas into a playable Roblox project.

## Core principle

A huge dream is not an MVP. The skill must push the user toward a playable vertical slice.

Bad MVP:

- open world;
- 30 islands;
- full economy;
- deep dialogue;
- PvP;
- bosses;
- crafting;
- pets;
- multiplayer raids;
- advanced sailing;
- all at once.

Good MVP:

- one island;
- one short quest;
- one enemy family;
- one simple reward;
- one ship prototype;
- one save loop;
- one HUD state;
- one analytics funnel.

## Design doc sections

- game promise;
- target player;
- platform/device assumptions;
- core loop;
- secondary loops;
- MVP scope;
- progression;
- economy;
- social/multiplayer;
- monetization, if any;
- safety/moderation;
- analytics KPIs;
- production milestones.

## Technical design sections

- architecture;
- services/controllers;
- data schema;
- remote contracts;
- UI states;
- performance budget;
- test plan;
- risks.

## Claude output requirements

When a user gives a big idea, Claude should produce:

1. vision summary;
2. MVP cut;
3. vertical slice plan;
4. systems list;
5. do-now/do-later split;
6. technical design;
7. first implementation milestone.
